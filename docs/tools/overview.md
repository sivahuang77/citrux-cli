# 🛠️ 內建工具箱 (Built-in Tools)

Citrux
CLI 將 AI 從「聊天機器人」轉變為「開發者夥伴」的關鍵，在於它所擁有的操作權力。

## 核心原子工具

### 1. `replace` (精密重構)

**描述**：在指定檔案中進行代碼段替換。

- **特性**：AI 會提供「更換前」與「更換後」的代碼塊。Citrux 會自動處理縮排、空行與分號，確保符合您的專案風格。
- **安全**：每次替換前，CLI 都會顯示 Diff 對比視窗供您審查。

### 2. `shell` (系統執行)

**描述**：在本地終端執行任何命令。

- **常見用法**：`npm install`, `go build`, `git commit`。
- **智能路徑**：AI 能自動識別專案根目錄，確保指令在正確的 CWD 下執行。

### 3. `web_search` (即時情報)

**描述**：透過 Google 或 Brave 進行網路檢索。

- **特性**：AI 在回答前會判斷是否需要查閱最新文件。它會彙整多個網頁的內容，並在答案中附上引用來源。

### 4. `ls`, `grep`, `glob` (文件索引)

**描述**：模仿 UNIX 開發習慣的文件檢索工具。

- **特性**：針對大型專案進行了優化，支援正則表達式，且能自動忽略 `.gitignore`
  中的檔案。

---

## 🛡️ 安全機制 (Safety & Permissions)

所有的工具調用都遵循 **「最小特權原則」**：

- **唯讀權限**：讀取檔案、清單目錄通常預設為自動核准。
- **寫入/執行權限**：涉及修改系統狀態的操作（如
  `shell`），Citrux 必須徵求您的明確同意（除非您在 `/settings`
  中開啟了 YOLO 模式）。
